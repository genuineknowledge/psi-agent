psi-agent 真实通道闭环测试结果

更新日期：2026-06-16 ｜ 记录范围：Telegram、Discord、Slack、WhatsApp

结论：Telegram、Discord、Slack 已经用测试账号跑通过“用户发消息 -> 平台事件 -> psi-agent -> 平台可见回复”的真实闭环。WhatsApp 因 Meta 开发者账号和号码验证问题暂时不继续投入。

注意：测试过程中不得把 token、secret、真实账号密码写入文档或提交到仓库。环境变量文件只保留在本地临时路径。

总览

平台 入站 出站 真实闭环状态 说明
Telegram Bot webhook，经公网 HTTPS 隧道转发到本机 Telegram Bot API sendMessage 已通过 需要设置 bot token、测试 chat id，并在测试前绑定 webhook URL。
Discord Discord Gateway MESSAGE_CREATE Discord REST API 发频道消息 已通过 Gateway 模式不需要公网 HTTPS。需要开启 message content intent，并把 bot 加进测试服务器/频道。
Slack Events API webhook，经公网 HTTPS 隧道转发到本机 Slack Web API chat.postMessage 已通过 需要 bot event、OAuth scope、重新安装应用、把 bot 邀请进测试频道。
WhatsApp WhatsApp Cloud API webhook Cloud API messages endpoint 暂缓 阻塞点是 Meta 开发者账号/号码验证，不是当前 psi-agent adapter 代码。

链路说明

用户在聊天软件发消息
  -> 平台 webhook 或 Gateway
  -> psi_agent.channel.platform 对应 adapter
  -> psi-agent session socket
  -> AI backend / model
  -> adapter.send_reply()
  -> 平台聊天窗口出现回复

已修正的测试点

  • 真实 channel smoke test 现在会等待 send_reply() 返回，不再只确认 webhook 入队。
  • 平台 webhook 处理采用异步入队返回，HTTP 响应为 {"ok": true, "messages": 1, "queued": 1, "duplicates": 0}。
  • 飞书、钉钉、微信桥、QQ桥的本地 mock E2E 已纳入单元测试。
  • psi-agent gateway 是通用 profile runtime，不引入任何外部项目专有概念。

重新执行真实测试

uv run pytest tests/integration/test_real_channels.py -v

执行前需要配置对应平台环境变量。Telegram、Slack、WhatsApp 的公网 HTTPS URL 变化后，平台控制台的 webhook URL 也要同步更新。
