psi-agent 通道接入与真实账号测试说明

更新日期：2026-06-16 ｜ 范围：Telegram、WhatsApp、Discord、Slack，以及国内通道的测试前置说明

当前结论：国外通道的协议 adapter、link parser、mock E2E 和真实账号测试入口已经接好。Telegram、Slack、Discord 已经做过真实账号闭环；WhatsApp 因 Meta 账号/号码验证暂时阻塞。

真实闭环定义：用户在聊天软件发消息，平台通过 webhook 或 Gateway 把事件送到 psi-agent，psi-agent 调用 session/model 生成回复，再通过平台 API 发回同一个聊天窗口。

代码入口

模块 职责
src/psi_agent/channel/link.py 解析和归一化 channel 链接，例如 t.me、slack://、feishu://、dingtalk://。不负责收发消息。
src/psi_agent/channel/platform.py 平台 adapter：验签、解析入站消息、调用 psi-agent session、发送平台回复。
src/psi_agent/gateway/profile.py 统一 profile gateway runtime：同时启动 AI backend、session server、多个 channel adapter，并写本 profile 的状态文件。

平台 入站方式 出站方式 状态

真实测试环境变量模板

真实测试默认跳过，只有设置 PSI_RUN_REAL_CHANNEL_TESTS=1 后才会调用外部平台 API。不要把真实 token 提交进仓库。

$env:PSI_RUN_REAL_CHANNEL_TESTS = "1"
$env:PUBLIC_WEBHOOK_BASE_URL = "https://your-public-https-domain.example"

$env:TELEGRAM_BOT_TOKEN = "..."
$env:TELEGRAM_TEST_CHAT_ID = "..."
$env:TELEGRAM_WEBHOOK_SECRET = "psi-agent-test-secret"

$env:SLACK_BOT_TOKEN = "..."
$env:SLACK_TEST_CHANNEL_ID = "..."
$env:SLACK_SIGNING_SECRET = "..."

$env:DISCORD_BOT_TOKEN = "..."
$env:DISCORD_TEST_CHANNEL_ID = "..."
$env:PSI_RUN_REAL_DISCORD_GATEWAY_TESTS = "1"
$env:DISCORD_GATEWAY_TEST_TEXT = "psi-agent gateway test"

$env:WHATSAPP_ACCESS_TOKEN = "..."
$env:WHATSAPP_PHONE_NUMBER_ID = "..."
$env:WHATSAPP_TEST_RECIPIENT = "..."
$env:WHATSAPP_VERIFY_TOKEN = "psi-agent-verify-token"

测试命令

uv run pytest tests/psi_agent/channel/test_link.py -q
uv run pytest tests/psi_agent/channel/test_platform.py -q
uv run pytest tests/integration/test_real_channels.py -v

当前的真实 smoke test 会等待平台 send_reply() 返回；不是只测 webhook 入队。因此如果 token、权限或平台 API 有问题，测试会失败或超时。

公网 HTTPS

Telegram、Slack、WhatsApp 的 webhook 需要公网 HTTPS 地址。本机开发可用 ngrok、Pinggy、Cloudflare Tunnel 等隧道工具把本地 8080 暴露出去。公网地址变化后，需要重新到平台控制台更新 webhook URL。

Discord Gateway 不需要公网 HTTPS，因为是 psi-agent 主动连 Discord Gateway。
