需要先澄清一个关键区分：**Coze** 实际上是字节跳动的一套产品，包含三层——SaaS 平台（国际版 coze.com / 国内版「扣子」coze.cn）、2025 年 7 月开源的 **Coze Studio**（自托管版本），以及配套的 **Coze Loop**（可观测/评测平台）。下面的分析会分别说明 SaaS 与开源版的差异。

## 架构设计与核心抽象

Coze 本质上是一个**可视化、低代码/无代码的 Agent 开发平台**，而非面向开发者的代码框架，编程模型以拖拽式画布为主。核心抽象包括：Bot/智能体、Workflow（节点式可视化工作流编排）、Plugin（工具）、Knowledge（知识库/RAG）、Memory（变量、数据库、长期记忆）、Card（UI 卡片）、Trigger（定时触发器）。智能体分**单 Agent 模式**和**多 Agent 模式**两种构建方式。开源的 Coze Studio 后端为 Golang 微服务、前端 React/TypeScript，底层编排能力建立在字节自家的 **Eino**（Go 语言 LLM 应用框架）之上，采用领域驱动设计，许可证为 Apache 2.0。

## 多智能体协作机制

多 Agent 协作主要通过**多 Agent 模式**实现：把多个子 Agent 作为节点，用「条件跳转」连接，本质是一个由控制流转移驱动的有限状态机/有向图（类似 supervisor 路由）。Workflow 内部也可放置多个 LLM 节点、子工作流节点串并联。状态共享依赖全局变量和数据库节点，消息以节点输入/输出形式传递。整体偏**确定性、流程编排型**，擅长可控的任务分解与路由，但在自由辩论、群聊式涌现协作（如 AutoGen 的 group chat）方面表达力较弱。

## 工具/插件生态

内置**插件商店**提供数百个官方与第三方插件（搜索、图像生成、各类 API 等），是 Coze 的强项之一。自定义工具支持三种方式：导入 OpenAPI/已有 API、在内置 IDE 写代码（Python/Node.js）、或封装为工作流。**MCP 协议**方面，国内「扣子」在 2025 年已支持接入 MCP Server（也能将自身能力暴露），但不同版本支持程度存在差异，建议核对当前版本。生态丰富度在国内同类产品中领先，背靠字节流量入口（抖音、飞书等发布渠道）。

## 部署与生产可用性

部署形态多样：SaaS 直接用 coze.com / coze.cn；自托管走开源 **Coze Studio**，官方提供 Docker Compose 一键部署。发布渠道覆盖 API、Web/Chat SDK、Discord、Telegram、飞书/Lark、微信等。可观测性由独立开源的 **Coze Loop（扣子罗盘）** 承担，提供 tracing、Prompt 管理、评测（eval）能力。企业级能力主要通过火山引擎商业化版本提供（团队空间、企业版）。容错/断点续跑等细节公开信息较少，自托管版的生产级 HA 方案需自行验证。

## 社区活跃度与文档质量

背靠字节跳动，迭代频繁。Coze Studio 自 2025 年 7 月开源后 GitHub Stars 增长很快（开源初期即达到万级量级，当前具体数字我无法联网确认，请以 GitHub 实时数据为准）。官方文档（coze.com/docs）较为完整，中英双语，配有大量图文教程；国内「扣子」的中文教程、案例和第三方课程非常丰富。社区渠道包括 Discord 及国内开发者社群。整体属于文档与教程资源充裕的一类。

## 与主流 LLM 的集成方式

国际版 coze.com 支持 GPT、Claude、Gemini 等主流模型；国内「扣子」以字节自家的**豆包（Doubao/Skylark）**为主，并接入 Kimi、DeepSeek 等国产模型。开源 Coze Studio 通过模型配置层（YAML 配置）接入，支持 OpenAI、Claude、火山方舟（Volcano Ark）、豆包、DeepSeek、Qwen、**Ollama 本地模型**等，因此自托管场景下可用本地模型。模型切换在 SaaS 端是下拉选择，在自托管端改配置文件，统一适配层依托 Eino 的 ChatModel 抽象。

---

**一句话定位**：Coze 是字节跳动出品、以可视化工作流编排为核心的 Agent 开发平台，最独特的价值在于把「低代码搭建 + 丰富插件生态 + 多渠道一键发布 + 可自托管开源（Coze Studio）」打包成一条完整链路，让非工程背景用户也能快速上线生产级 AI 智能体。
