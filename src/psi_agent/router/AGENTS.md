# Semantic Router Layer

The router is an optional AI-compatible service placed between Session and a
set of candidate upstream model services. It performs exactly one semantic
selection request per routable Chat Completions request and then proxies the
original request to one selected upstream.

## Security boundary

The routing model may see only:

- the bounded, sanitized conversation context;
- an opaque zero-based candidate index; and
- each candidate's `description`.

Never include candidate `model_name`, candidate `addr`, `default_addr`, or API
keys in the routing prompt. A routing response selects an integer index only;
all model/address lookup stays in validated local configuration. Do not accept
an address or model name generated by the routing model.

## Request behavior

- Semantic success copies the original request and overwrites only `model`
  with the selected upstream's `model_name`.
- Routing failure, timeout, malformed output, and missing user context use
  `default_addr` and preserve the original request's `model`.
- A default-upstream failure is terminal. Never recursively fall back.
- Unknown Chat Completions fields are passed through unchanged.
- Normal upstream SSE is proxied as bytes. Do not reconstruct content,
  reasoning, tool-call, extension, or `[DONE]` chunks.
- Failures after HTTP 200 preparation use the internal
  `finish_reason="error"` chunk.

## Context handling

`selector.serialize_context()` keeps system, user, and assistant text, keeps
assistant tool names without argument bodies, replaces non-text media with
short markers, and omits tool result bodies. It preserves the first system
message and newest useful conversation blocks within `router_context_chars`.
No usable user content means direct default fallback.

## Lifecycle and logging

All I/O uses aiohttp and anyio. Network responses and sessions must remain in
async context managers so cancellation closes both routing-model and business
upstream connections. aiohttp runner cleanup is shielded on startup failure and
shutdown. `AiRouter.run()` must call `setup_logging(verbose=self.verbose)` as
its first executable statement.

Every proxied SSE chunk is DEBUG. Startup, shutdown, and selected destination
are INFO. Recoverable semantic-selection failures are WARNING. Proxy failures
are ERROR. With `--log-router-details`, the selection summary contains only
`Router reason` and the final `Router result` (`model_name` and `addr`). Do not
add candidate descriptions, indices, context lengths, source labels, raw
context, or API keys to these summary logs.

## CLI schema

The command is `psi-agent ai router`. Each repeated `--upstream` value is
strict JSON containing exactly `model_name`, `addr`, and `description`.
`--default-addr` is a direct service address and need not appear in the
candidate list. Router model configuration may fall back to
`PSI_ROUTER_MODEL`, `PSI_ROUTER_BASE_URL`, and `PSI_ROUTER_API_KEY`.
