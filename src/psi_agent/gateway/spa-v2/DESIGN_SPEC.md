# HaiTun Agent 任务工作台开发实现规范

## 1. 产品结构

HaiTun Agent 的主工作区不是传统列表，而是一组可切换的任务单元。每个任务单元必须同时包含：

1. 当前任务卡片或任务总览卡片。
2. 卡片分页与当前位置。
3. 与当前卡片严格对应的对话。

任何切换方式都必须让三个部分作为同一个整体进出，不能只替换卡片、让对话内容瞬间跳变。

主导航固定为：

- 任务总览
- 历史任务
- 任务模板

新建任务是独立的中央对话工作区，不是弹窗。任务模板是独立模板库，进入模板库后才能新建模板或选择使用模板。

## 2. 统一词典

| 对象 | 唯一名称 | 禁用名称 |
| --- | --- | --- |
| 全局总览 | 任务总览 | 总览、工作总览、Summary |
| 需要用户介入 | 待您处理 | 等您处理、需要您确认（作为状态名） |
| 新产生的文件或物料 | 新交付物 | 待验收、可以验收 |
| 文件查看入口 | 查看交付物 | 验收任务 |
| 文件保存动作 | 保存到成果库 | 验收并归档 |
| 任务结束 | 已完成 | 已验收 |
| 普通运行任务 | 进行中 | 运行中、推进中混用 |
| 长期任务 | 持续运行 | 一直转、无限任务 |

所有面向用户的第二人称统一为“您”。无障碍标签、占位文字、通知、Toast 和 Agent 回复同样遵循该规则。

## 3. 状态模型

任务状态与交付物状态必须正交，不允许通过一个字段推导另一个字段。

```ts
type TaskStatus =
  | "working"      // 进行中
  | "attention"    // 待您处理
  | "completed"    // 已完成
  | "continuous";  // 持续运行

type DeliveryState =
  | "none"         // 无文件交付物
  | "generating"   // 交付物生成中
  | "ready"        // 新交付物可查看
  | "saved";       // 已保存到成果库
```

关键业务规则：

- 进行中的任务可以有 `deliveryState="ready"`，代表阶段性交付物已经产生。
- 已完成的任务可以有 `deliveryState="none"`，代表任务通过系统状态更新、通知或其他非文件结果完成。
- 打开或保存交付物只更新 `deliveryState`，不能自动把任务改为 `completed`。
- “让 Agent 修改”会建立新一轮执行，因此可以把任务改为 `working`，并把交付物改为 `generating`；这是显式业务动作，不是状态耦合。
- 持续任务的百分比表示“本轮巡检进度”，不是整个任务的完成度。
- 任务总览的综合进度计算必须排除持续任务，避免把一轮巡检误当作总任务进度。

Demo 中覆盖四个必要示例：

| 示例 | 任务状态 | 交付物状态 | 用途 |
| --- | --- | --- | --- |
| 灰度发布方案 | 待您处理，72% | 生成中 | 用户介入后继续执行 |
| 市场情报周报 | 已完成，100% | 新交付物 | 任务完成且有文件 |
| 用户反馈分析 | 进行中，46% | 新交付物 | 任务未完成但有阶段物料 |
| 例会行动项同步 | 已完成，100% | 无 | 任务完成但没有文件 |
| 竞品动态监测 | 持续运行，本轮 64% | 无 | 长期任务的周期进度 |

## 4. 图标与材质

| 语义 | 视觉组件 | 色彩 |
| --- | --- | --- |
| 进行中 | 蓝色圆形进度环 | 品牌蓝 |
| 持续运行 | 同一进度环，中心时钟，本轮真实百分比 | 品牌蓝 |
| 待您处理 | 圆形叹号 | 红色 |
| 已完成 | 圆形对勾 | 成功绿 |
| 交付物生成中 | 灰色宝箱 | 中性灰 |
| 新交付物 | 金色发光宝箱 | 金色 |
| 已保存交付物 | 金色宝箱 + 对勾 | 金色，降低呼吸动效 |

宝箱必须复用同一 `TreasureVisual` 结构。左上角入口、任务列表、任务卡、任务总览指标、收件箱、成果库按钮和金币庆祝都不能另画一套箱体。

任务完成不能使用宝箱，交付物也不能使用完成对勾代替。

## 5. 品牌视觉

主色为 HaiTun 蓝：

```css
--brand-500: #007bff; /* rgb(0, 123, 255) */
--brand-400: #2e9eff;
--brand-600: #006ae6;
--brand-700: #0057bd;
--navy-950: #06172b;
--canvas: #f2f7fd;
```

使用规则：

- 蓝色：品牌、主要按钮、运行状态、选中态、焦点态和进度。
- 红色：仅表示需要用户处理。
- 金色：仅表示新交付物或成果奖励。
- 绿色：仅表示任务完成或操作成功，不能承担品牌强调。
- 海豚 Logo 统一使用 `public/haitun-dolphin.png`，覆盖侧栏、新建任务页、Agent 对话头像和浏览器图标；页面内必须直接读取静态资源，不能依赖运行时图片优化路由。
- 正文使用系统中文字体栈，发布产物不得包含开发机绝对字体路径。

## 6. 桌面端布局

- 左侧栏展开宽度：292px；小屏桌面可缩至 260px。
- 左侧栏完全收起后主区域占满宽度。
- 桌面端展开与收起复用同一个固定圆形控件：纵向位置、尺寸和材质保持不变，只沿侧栏边缘横向移动并切换图标；任务总览、新建任务、任务模板三个页面始终可见。
- 移动端所有主页面始终保留左上角侧栏入口；新建任务、任务模板页同时显示返回入口。
- 主区域顶部右侧始终保留“新建任务”和收件箱。
- 主卡片宽度约 570–820px，高度 `clamp(420px, 57dvh, 560px)`。
- 左右箭头、键盘方向键、分页点和鼠标拖拽调用同一个 `goTo()` 切换逻辑。

## 7. 移动端布局

- 响应式断点：880px。
- 侧栏使用抽屉，宽度 `min(88vw, 330px)`。
- 卡片支持横向手势，纵向手势继续交给页面和对话。
- 顶部始终保留侧栏入口与新建任务入口。
- 从新建任务和模板库可通过页面左上角返回，也可通过页面底部返回任务总览。
- 宝箱打开时，在支持振动的粗指针设备触发 12ms 轻震；保存成果时触发 `[10, 28, 16]` 节奏。

## 8. 卡片与对话整体切换

DOM 结构必须保持：

```text
card-chat-unit-layer
└── card-chat-unit
    ├── task-context-stack
    │   ├── card-transition-frame
    │   │   ├── card-swipe-surface → task-card / overview-card
    │   │   └── compact-card-layer → compact-context-card
    │   └── card-pagination
    └── context-chat
        ├── chat-context-row
        ├── focus-workspace-scroll
        │   ├── 当前状态与执行路径
        │   ├── 任务历史
        │   └── 历史交付物
        └── composer
```

切换时同时渲染旧层和新层：

- 旧层：260ms 离场，位移 70px、轻微旋转和模糊。
- 新层：延迟 55ms，410ms 入场，位移 96px。
- 整体过渡窗口：470ms。
- 统一缓动：`cubic-bezier(0.16, 1, 0.3, 1)`。
- 减少动态效果系统偏好开启时，动画应降至近乎即时。

拖拽位移施加到整个任务单元，但手势只能从卡片区域开始。`button/input/textarea/a` 必须排除，避免点击宝箱或输入框时误触切卡。

每张卡片的对话草稿按 `cardId` 独立保存，切换任务后不能把 A 任务的未发送草稿带到 B 任务。

## 9. 对话聚焦模式

输入框获得焦点后进入显式对话聚焦模式：

- 桌面端把整个任务单元改为左右双栏：左侧紧凑上下文卡约占 32%，右侧任务工作区约占 68%；聚焦轨最大宽度 1040px。
- 原卡不能被降低透明度后裁切。原卡内容先淡出，卡体在 420ms 内收窄，紧凑上下文卡延迟接管；收起时按相反方向还原。
- 紧凑上下文卡必须完整保留类别、状态、标题、摘要、进度、预计完成时间、最近更新时间、全部执行步骤和交付物状态，信息只能重排，不能删除。
- 移动端改为上下布局：紧凑上下文卡高度 190–258px，对话工作区占据余下空间；短屏时紧凑卡自身可滚动，确保全部原卡信息可访问。
- 右侧/下方任务工作区展示当前状态、执行路径、任务历史和历史交付物；任务通知和最近沟通合并到任务历史中。
- `ready/saved` 交付物以文件卡在页面内渲染，可继续打开完整预览；`generating` 只能标为“预计交付/生成中”，不能计入历史交付物。
- 任务历史与交付物区域独立滚动，输入框固定在工作区底部，虚拟键盘出现时仍应可见。
- 提供明确的“收起”按钮；Esc 也可退出。
- 收起时主动让输入框失焦，但保留该任务的未发送草稿。
- 切换卡片时自动退出聚焦模式；离场层必须保留切换前的聚焦几何状态，避免先恢复成大卡再离场。
- 当前卡片的 React key 只绑定 `cardId`，不能绑定过渡 token，避免动画结束时输入框 remount、光标或中文输入法状态丢失。

## 10. 全局搜索

入口位于侧栏“新建任务”按钮正下方，快捷键为 `Cmd/Ctrl + K`。

检索字段：

- 历史任务：完整标题、短标题、分类、摘要、状态、交付物名称。
- 任务模板：名称、分类、描述、默认任务描述、交付物名称。

结果分为“历史任务”和“任务模板”两组：

- 点击历史任务：打开对应任务卡片。
- 点击任务模板：进入模板库并使用模板名预筛选；不能直接创建任务。
- 无结果时显示明确空状态。

## 11. 收件箱

PC 端收件箱使用右侧抽屉，包含：

- 待您处理：红色叹号，点击进入任务。
- 新交付物：金色宝箱，点击直接打开交付物预览。
- 运行更新：蓝色进度环，点击进入对应任务。

未读数动态计算；支持单条标记已读、全部标为已读、遮罩关闭和 Esc 关闭。打开交付物前必须先关闭收件箱，避免两个右侧抽屉叠加。

## 12. 侧栏设置

账户区三点菜单包含：

- 通知与提醒开关。
- 动效与触觉反馈开关。
- 默认交付位置：成果库。
- 语言与称呼：简体中文 · 您。
- 键盘快捷键。
- 帮助与反馈。
- 产品标识：HaiTun Agent。

开关必须真实影响演示：通知开关控制未读角标；动效与触觉开关同时关闭宝箱循环动效、庆祝动效和移动端振动。

## 13. 新建任务与模板

- `Cmd/Ctrl + N` 从任意主页面进入新建任务对话。
- 新建任务首屏包含海豚 Logo、“有什么可以帮您”、预设任务类型和中央输入框。
- 发送首条描述后创建任务，并**自动进入该任务的对话分屏聚焦**（左上下文 / 右对话），不再停在新建页本地气泡。
- 附件入口必须给出已选择文件反馈。
- 模板库默认展示沉淀模板；“新建模板”只能在模板库内部出现。
- “使用模板”只预填新建任务对话，不能静默创建任务。

## 14. 交付物奖励反馈

- 灰色宝箱不可点击，Title 为“交付物生成中”。
- 金色宝箱以轻微呼吸和星光提示可领取。
- 点击宝箱先播放 430ms 开箱与 9 枚金币飞散，再打开右侧预览。
- “保存到成果库”播放更大范围金币庆祝，约 820ms 后写入 `deliveryState="saved"`。
- 保存后任务状态不变。
- 设置关闭动效或系统开启 `prefers-reduced-motion` 时，保留状态反馈但缩短或取消运动。

## 15. 无障碍与键盘

- 所有纯图标按钮必须有中文 `aria-label`。
- 抽屉使用 `role="dialog"` 和 `aria-modal="true"`。
- 侧栏设置使用 `aria-haspopup="menu"` 与 `aria-expanded`。
- Enter/Space 可打开侧栏任务条。
- 左右方向键切换任务单元；Esc 按层级关闭交付物、收件箱、设置、搜索、对话聚焦和侧栏。
- 输入框或文本域内的普通方向键不得触发切卡。
- 移动端输入框与文本域字号不得低于 16px，避免 iOS 聚焦时自动缩放页面。

## 16. 开发验收用例

1. 全仓用户可见文案检索不到“你”，只能使用“您”。
2. 任务总览在侧栏、卡片、顶栏、返回按钮和对话标题中名称完全一致。
3. 用户反馈分析在 46% 时仍能打开金色阶段交付物，保存后任务仍为进行中。
4. 例会行动项同步显示已完成，但不出现宝箱。
5. 持续任务显示“本轮巡检进度 64%”，不显示任务完成倒计时。
6. 用箭头、键盘、分页点和拖拽分别切换一次，卡片与对话必须同步进出。
7. 在两个任务中分别输入未发送草稿，来回切换后草稿不能串任务。
8. 在任务模板页收起侧栏，页面左上角仍可展开侧栏。
9. 全局搜索可以分别命中一个历史任务和一个模板；模板结果只进入模板库。
10. 收件箱三类通知均可点击，未读数随操作变化。
11. 关闭动效与触觉反馈后，宝箱不再呼吸，手机不再振动。
12. 桌面与移动端输入任务对话时，卡片缩小、上下文详情出现、对话历史展开。
13. `npm run lint`、`npm run build` 和 `npm test` 全部通过。
14. 页面内海豚 Logo、侧栏深色背景和浅色文字在刷新后仍正常，网络请求中没有图片优化路由 4xx。
15. 桌面端展开和收起侧栏前后，开关的纵坐标、尺寸与材质完全一致。
16. 900px 宽桌面与 390px 宽手机均无横向溢出、卡片裁切或顶栏遮挡。
17. 收件箱可打开；交付物下载有明确反馈；保存成果时显示宝箱开启和金币洒落动效。
